# 書籍管理アプリ

新入社員向け **バグ探しトレーニング教材（第2弾 / JavaScript編）** です。  
意図的にバグが仕込まれた書籍管理アプリを動かしながら、仕様との差異を発見してください。

---

## 実行方法

`index.html` をブラウザで開くだけで動作します。サーバー不要。

```
kensyu-bug2/
├── index.html   # HTML
├── style.css    # スタイルシート
├── app.js       # ロジック（JavaScript）
└── README.md    # 本ドキュメント
```

> ⚠️ 必ず **ブラウザの開発者ツール（F12）の Console タブを開いた状態** で作業してください。  
> エラーの多くはコンソールに手がかりが出ています。

## 機能仕様

### 画面構成

| 要素 | 説明 |
|---|---|
| タイトル | 「📚 書籍管理アプリ」 |
| 検索バー | キーワード入力欄・検索ボタン・リセットボタン |
| 書籍一覧 | サーバーから取得した書籍をカード形式で一覧表示 |
| カート | 追加した書籍の一覧と合計金額を表示 |

---

### 書籍データの取得（非同期）

`app.js` 内の `fetchBooks()` 関数が、API 呼び出しを模した非同期処理として書籍データを返します。

| 仕様 | 内容 |
|---|---|
| 取得方法 | `Promise` を返す関数 `fetchBooks()` を呼び出す |
| 遅延 | 300ms |
| 失敗確率 | **30% の確率でエラーを返す**（実APIの不安定さを模擬） |
| 失敗時の挙動 | 画面に **エラーメッセージを表示** する（画面が真っ白にならないこと） |

> `fetchBooks()` の中身は **書き換え禁止** です。呼び出し側を正しく実装してください。

---

### 書籍カードの表示

書籍一覧の各カードには以下の情報を表示します。

| 項目 | 表示形式 | 備考 |
|---|---|---|
| タイトル | そのまま表示 | |
| 著者 | `著者: ◯◯` | |
| 価格 | `◯◯円` | |
| 在庫 | `在庫: ◯冊` | **在庫が 0 のときも「0冊」と表示** すること |
| カート追加ボタン | `カートに追加` | **在庫が 0 のときは disabled（押せない）** |

#### 在庫切れ時のスタイル

- 在庫が `0` のときは在庫表示を **赤色（`stock-out` クラス）** で表示する
- 在庫が `0` のときはカート追加ボタンを **非活性（disabled）** にする

> **ポイント**  
> - 在庫 `0` は「未取得（不明）」ではなく **「在庫切れ」** という有効な値です。  
> - `undefined` の場合と `0` の場合を **区別** してください。

---

### 検索機能

検索バーにキーワードを入力して検索ボタンを押すと、書籍一覧が絞り込まれます。

| 条件 | 挙動 |
|---|---|
| キーワードが **タイトル または 著者** に含まれる | 表示する |
| 上記いずれにも含まれない | 表示しない |
| キーワードが空 | 全件表示 |
| 該当なし | 「書籍が見つかりませんでした」と表示 |
| リセットボタン押下 | 入力欄をクリアし、全件表示に戻す |

---

### カート機能

| 操作 | 挙動 |
|---|---|
| 「カートに追加」ボタン押下 | その書籍をカートに追加する |
| カート内「削除」ボタン押下 | **その 1 件だけ** を削除する（他の項目には影響しない） |
| 合計金額 | カート内のすべての書籍の `price` の合計を表示 |
| カートが空 | 「カートは空です」と表示し、合計は `0 円` |

## バグ探しチャレンジ

このアプリには **合計 8 つのバグ** が意図的に仕込まれています。  
上記の仕様を正として、`app.js` の中の実装の誤りをすべて見つけてください。

> `fetchBooks()` 関数の中身（モックAPI部分）には **バグはありません**。  
> その下のアプリ本体にすべて潜んでいます。

### バグの分類

| カテゴリ | 件数 | ヒント |
|---|---|---|
| 非同期処理（`async` / `await`） | 2 件 | コンソールのエラーと、何度かリロードしたときの挙動を確認しましょう |
| 配列操作（`filter` / `map` / `splice`） | 3 件 | 検索結果・カードの表示・カートの削除を、それぞれ実際に試してみましょう |
| ES6 構文（分割代入 / テンプレートリテラル） | 2 件 | 画面に `undefined` や `${...}` といった文字が出ていないか観察しましょう |
| フォールバック（デフォルト値の扱い） | 1 件 | 在庫が `0` の書籍の表示が仕様どおりになっているか確認しましょう |

### 取り組み方

1. ブラウザで `index.html` を開く
2. 仕様どおりに動作しない箇所を探す
3. `app.js` のコードを読み、原因を特定する
4. 修正して正しく動作することを確認する

> 開発者ツールの **Console タブ** を常に開いて作業してください。

### API失敗時の挙動

ページを何度かリロードすると、約 30% の確率で書籍取得に失敗します。  
失敗時には画面に「⚠️ サーバーエラー: 書籍データの取得に失敗しました」のようなエラーメッセージが表示されればOKです。  
（画面が「読み込み中...」のまま固まったり、真っ白になったりするのはバグです）
